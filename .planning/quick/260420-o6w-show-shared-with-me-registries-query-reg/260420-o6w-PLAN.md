<objective>
Make registries shared with the current user (via invite) appear on their Home screen alongside registries they own.

**Purpose:** Currently `FirestoreDataSource.observeRegistries(uid)` only queries `.whereEqualTo("ownerId", uid)`, so invited users never see registries shared with them. Reported symptom: "Victor doesn't see Maria's registry" after Maria's invite processed successfully. The invite pipeline already writes `invitedUsers.{uid} = true` on the registry doc (see `functions/src/registry/inviteToRegistry.ts` L85-L91) and `firestore.rules` already permits invited users to read the doc (see `isInvited()` at L30-L36) — the only missing piece is a client-side query that actually fetches invited registries.

**Output:** A second Firestore snapshot query plus a merge/dedupe/sort pipeline in the repository, so the existing `RegistryListViewModel` surfaces owned + invited registries in a single list. No rules change. No UI change.
</objective>

      ```kotlin
      override fun observeRegistries(ownerId: String): Flow<List<Registry>> {
          // Parameter name kept as `ownerId` for interface stability; semantically
          // this is the current user's UID. Result includes registries the user
          // owns AND registries where invitedUsers[uid] == true.
          val owned = dataSource.observeOwnedRegistries(ownerId)
          val invited = dataSource.observeInvitedRegistries(ownerId)
          return combine(owned, invited) { ownedDtos, invitedDtos ->
              // Dedupe by id — owner wins if a user is both owner and self-invited.
              val byId = linkedMapOf<String, RegistryDto>()
              for (dto in ownedDtos) byId[dto.id] = dto
              for (dto in invitedDtos) byId.putIfAbsent(dto.id, dto)
              byId.values
                  .map { it.toDomain() }
                  .sortedByDescending { it.updatedAt }
          }
      }
      ```
