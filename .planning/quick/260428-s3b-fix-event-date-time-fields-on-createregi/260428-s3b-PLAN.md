<objective>
Fix the broken Event Date and Event Time fields on the Create / Edit Registry form so the date picker actually opens on tap and the time field becomes a working time picker that encodes hour/minute into the existing `eventDateMs: Long?` domain field.

Purpose: Without these fields, registry owners cannot record event time at all (time is an inert stub) and date entry only works by accident on devices where ripple/focus quirks let the clickable through. The reservation-flow value proposition — knowing WHEN the gift is needed by — depends on accurate event timestamps.

Output: Two files touched (`CreateRegistryViewModel.kt`, `CreateRegistryScreen.kt`), no domain or persistence changes (the existing `eventDateMs: Long` is sufficient — hour/minute are encoded into the same Long).

<discovery_notes>
Discovery Level 0: pure internal work using established Compose + Kotlin patterns.
- `MutableInteractionSource` + `LaunchedEffect` collecting `PressInteraction.Release` is the documented Compose pattern for "TextField as button" (used in Material3 docs and many community samples). Verified by the bug report's own analysis.
- `android.app.TimePickerDialog` is the framework parallel of the existing `android.app.DatePickerDialog` already used in this file. No new library.
- `android.text.format.DateFormat.is24HourFormat(context)` is the locale-aware 24h check (vs. parsing `Locale` directly).
- No new dependencies; no new strings; no domain change. All gradle config untouched.

DISABLED-FIELD INTERACTION SOURCE NOTE (per bug report constraint): Compose's `OutlinedTextField` with `enabled = false` blocks pointer events at the focusable layer — `PressInteraction.Release` is NOT emitted on disabled fields. The Time field MUST therefore stay `enabled = true` AND use `enabled = (eventDateMs != null)` as a state-driven guard inside the LaunchedEffect collector body (see Task 2). Visual disabled appearance is achieved via `disabledTextColor`/`disabledBorderColor` already wired in `giftMaisonFieldColors()`, which we replicate here by checking the gate inside the collector and short-circuiting before showing the picker. This is simpler than the "Box overlay matchParentSize" fallback and behaves identically.
</discovery_notes>
