## Spacing Scale

GiftMaison spacing tokens (Phase 8) are the spacing system. Phase 10 uses only named tokens — no raw `dp` literals on new composables.

| Token | Value | Phase 10 Usage |
|-------|-------|----------------|
| `spacing.gap4` | 4 dp | Bullet separators in stats line, icon-to-text micro gaps |
| `spacing.gap8` | 8 dp | Intra-field group gaps, divider margin, card-section spacing |
| `spacing.gap10` | 10 dp | Inter-card vertical gap in LazyColumn |
| `spacing.gap12` | 12 dp | Vertical gap between form field pairs on Auth screen |
| `spacing.gap14` | 14 dp | Vertical gap between field groups (first/last name → email) |
| `spacing.gap16` | 16 dp | Section-to-section vertical gap on Auth; default padding |
| `spacing.gap20` | 20 dp | Hero bottom padding; Google banner internal padding |
| `spacing.edge` | 16 dp | Screen horizontal edge padding (Auth content, card horizontal) |
| `spacing.edgeWide` | 20 dp | Wider horizontal edge (Google banner side padding per handoff) |

**Touch-target exception:** Avatar button on Home top bar — minimum 44 dp tap target (`Modifier.size(44.dp)` wrapping 30 dp circle). Consistent with Android accessibility minimum.

## Screen 06 — Onboarding / Auth Screen Contract

### Layout

Full-screen `Column` with `verticalScroll`. Background: `colors.paper`. No `Scaffold` top bar (wordmark is inline, not in a Material3 TopAppBar).

```
┌─────────────────────────────────┐
│  [wordmark]          spacing.edge horizontal   │
│                                 │
│  "Start your          displayL italic inkSoft  │
│   first registry."    displayL italic + accent period │
│                                 │
│  [Google banner]      radius16, googleBannerShadow │
│                                 │
│  ─── or sign up with email ───  │
│                                 │
│  [First name field]   (sign-up mode only)      │
│  [Last name field]    (sign-up mode only)      │
│  [Email field]        accent caret on focus    │
│  [Password field]     eye-toggle trailing icon │
│  [Confirm pwd field]  (sign-up mode only)      │
│                                 │
│  [Sign up CTA]        ink pill, full-width     │
│                                 │
│  Terms line                     │
│                                 │
│  [Log in footer pill] ghost border, centered   │
└─────────────────────────────────┘
```

### Top Bar (inline, not TopAppBar)

- `GiftMaisonWordmark(fontSize = 20.sp)` left-aligned
- Padding: `spacing.edge` horizontal, `spacing.gap16` vertical top
- No back arrow, no actions

### Headline

```kotlin
Column(modifier = Modifier.padding(horizontal = spacing.edge)) {
    Text("Start your", style = typography.displayL, fontStyle = Italic, color = colors.inkSoft)
    // Second line as AnnotatedString: "first registry" in colors.ink + "." in colors.accent
    Text(annotated("first registry."), style = typography.displayL, fontStyle = Italic)
}
```

Vertical gap before Google banner: `spacing.gap20`

### Google Banner

Shape: `shapes.radius16`. Shadow: `Modifier.googleBannerShadow(colors.accent)`.
Background: `colors.paperDeep`. Padding: `spacing.gap14` vertical, `spacing.edgeWide` horizontal.

Internal layout:
- Left: 20×20 dp white circle with Google "G" icon (Material Icons: `Icons.Default.`... or a drawn `Canvas` circle). Background white, icon `#4285F4`/`#EA4335`/`#FBBC05`/`#34A853` sectors — or use a single-colour "G" in `colors.ink` as acceptable placeholder.
- Centre: `typography.bodyL` "Continue with Google" in `colors.ink`
- Right: `→` symbol in `FontStyle.Italic`, `InstrumentSerifFamily`, 18.sp, `colors.accent`
- Top-right corner decoration: 3 concentric circles drawn with `Canvas` (or layered `Box` circles using `Modifier.border`):
  - Outer ring: ~60 dp diameter, `colors.accentInk` at alpha 0.08, stroke 1 dp
  - Middle ring: ~40 dp, alpha 0.12, stroke 1 dp
  - Inner ring: ~24 dp, alpha 0.18, stroke 1 dp
  - Positioned with `Modifier.align(Alignment.TopEnd).offset(x = 8.dp, y = (-8).dp)` or `Canvas` absolute coords

Full banner is tappable (`Modifier.clickable`), calls existing `viewModel.signInWithGoogle()`.

### Divider

`Row` with `HorizontalDivider` + `colors.line`, centred `typography.bodyS` "or sign up with email" in `colors.inkFaint`.

### Input Fields

Field style: `OutlinedTextField`, `shape = shapes.radius12`, `colors = TextFieldDefaults.outlinedTextFieldColors` with:
- `containerColor = colors.paperDeep`
- `focusedBorderColor = colors.accent`
- `unfocusedBorderColor = colors.line`
- `cursorColor = colors.accent`
- `focusedLabelColor = colors.accent`
- `unfocusedLabelColor = colors.inkFaint`
- `placeholderColor = colors.inkFaint`
- `textColor = colors.ink`

Fields in sign-up mode (in order): First name, Last name, Email, Password, Confirm password.
Fields in sign-in mode: Email, Password.

First name + Last name in sign-up: placed in a `Row` with `spacing.gap8` gap between them (each `Modifier.weight(1f)`), matching handoff 2-column layout. Gap above: `spacing.gap12`.

Email field: `KeyboardType.Email`, `ImeAction.Next`. No leading icon (handoff has none — drop the existing `Icons.Default.Email` leading icon).

Password field: trailing eye-toggle `IconButton` (existing implementation kept). No leading icon.

Confirm password: sign-up mode only, same style as password field.

### Sign Up CTA

Shape: `shapes.pill`. Background: `colors.ink`. Text: `typography.bodyL`, `colors.paper`, "Sign up" (sign-up mode) / "Log in" (sign-in mode).

Height: 52 dp (full-width). Gap above: `spacing.gap20`. Loading state: `CircularProgressIndicator` centred, `colors.paper`, size 20 dp.

### Terms Line

Centred single line, `typography.bodyXS`, `colors.inkFaint`:
`"By signing up you agree to our [Terms] and [Privacy Policy]"`
Terms and Privacy Policy as tappable spans using `AnnotatedString` with `StringAnnotation`, each in `colors.accent`, `TextDecoration.Underline`. Opens `Intent.ACTION_VIEW` to placeholder URLs from string keys `auth_terms_url` / `auth_privacy_url`. Gap above: `spacing.gap12`.

### Footer "Log in" Pill

Centred, `shapes.pill` border, `colors.line` stroke 1 dp, transparent background.
Label: `typography.bodyMEmphasis`, `colors.ink`, "Already have an account? Log in"
Alternatively: separate `Text("Already have an account?")` + underlined `TextButton("Log in")` in a `Row` — Claude's choice per CONTEXT.md.
Gap above: `spacing.gap16`. Bottom screen padding: `spacing.gap20`.

### Mode Toggle Behaviour

Footer pill / "Log in" button calls `isSignUpMode = false` (and `viewModel.clearError()`). Header footer text switches via existing `auth_have_account_prompt` / `auth_no_account_prompt` keys. Default mode: sign-up (`isSignUpMode = true` as initial state — matches CONTEXT.md decision).

## Screen 07 — Home / All Registries Screen Contract

### Layout

`Scaffold` with custom top bar (inline `Row`, not `TopAppBar`). Background: `colors.paper`. Bottom padding accounts for Phase 9 `GiftMaisonBottomNav` height (already handled by Scaffold + nav padding in AppNavigation).

```
┌─────────────────────────────────┐
│  [wordmark]          [avatar]   │  top bar inline Row
│                                 │
│  Your registries      displayXL │
│  3 active · 12 items total  monoCaps inkFaint │
│                                 │
│  [Active][Drafts][Past]  segmented tabs │
│                                 │
│  ┌──────────────────────┐       │
│  │ [16:9 hero image]    │  PRIMARY card (ink bg) │
│  │ [occasion pill] [date]│       │
│  │ Registry title        │       │
│  │ N items · M reserved  │       │
│  └──────────────────────┘       │
│                                 │
│  ┌──────────────────────┐       │
│  │ [16:9 hero image]    │  secondary card │
│  └──────────────────────┘       │
│  ...                            │
└─────────────────────────────────┘
```

### Top Bar (inline Row, not TopAppBar)

Full-width `Row`, `Modifier.padding(horizontal = spacing.edge, vertical = spacing.gap16)`, `verticalAlignment = Alignment.CenterVertically`.

- Left: `GiftMaisonWordmark(fontSize = 20.sp)`
- Spacer `Modifier.weight(1f)`
- Right: `AvatarButton` composable (30 dp circle, `colors.second` background, initials `typography.bodyMEmphasis` `colors.paper`, tappable at min 44 dp, navigates to `SettingsKey`)

Avatar initials derivation: `displayName.split(" ").take(2).map { it.first().uppercaseChar() }.joinToString("")`. Fallback: first char of email. Final fallback: `"?"`.

### Headline + Caption

`Column`, `Modifier.padding(horizontal = spacing.edge)`:
- `Text("Your registries", style = typography.displayXL, color = colors.ink)`
- Gap: `spacing.gap4`
- `Text("N active · M items total", style = typography.monoCaps, color = colors.inkFaint)` — stats derived client-side from filtered lists counts

Gap below: `spacing.gap16`

### Segmented Tabs

Custom `SegmentedTabs` composable (NOT Material3 `SegmentedButton`). Lives in `ui/registry/list/SegmentedTabs.kt` or `ui/common/SegmentedTabs.kt` — executor's choice.

Track: full-width pill shape (`shapes.pill`), `colors.paperDeep` background, `spacing.gap8` vertical padding, `spacing.edge` horizontal padding on the outer container.

Each tab: equal-width `Box(Modifier.weight(1f))`, centred label `typography.monoCaps`.

Selected state: white (`colors.paper`) pill with `colors.line` drop shadow at 2 dp elevation drawn behind the pill. Label: `colors.ink`. Animated: `animateColorAsState` for background, no slide animation needed.

Unselected state: transparent background. Label: `colors.inkFaint`.

Tabs: "Active" / "Drafts" / "Past". Keys: `home_tab_active`, `home_tab_drafts`, `home_tab_past`.

Tab state: `var selectedTab by rememberSaveable { mutableStateOf(0) }` — persists rotation, resets on cold start (CONTEXT.md).

### Tab Filtering Logic (client-side, no ViewModel change)

```
Active  = eventDateMs == null || eventDateMs >= startOfTodayMs
Past    = eventDateMs != null && eventDateMs < startOfTodayMs
Drafts  = title.isBlank() || items.isEmpty()  // heuristic, CONTEXT.md approved
```

`startOfTodayMs` computed once per composition: `LocalDate.now().atStartOfDay().toEpochMilli()` (Java 8 time API, available on API 26+; use `Calendar.getInstance()` for minSdk 23 compat).

### Per-Tab Empty State

When filtered list is empty: centred `Text(style = typography.bodyM, color = colors.inkFaint)` with `spacing.gap20` bottom padding. No illustration, no CTA (CONTEXT.md + REQUIREMENTS.md § Out of Scope).

### Registry Card

Shape: `shapes.radius16`. Horizontal edge: `spacing.edge`. Inter-card gap: `spacing.gap10`.

**Primary card** (`isPrimary = registries.maxByOrNull { it.updatedAt }?.id == registry.id`):
- Background: `colors.ink`
- Image: `AsyncImage(Coil)`, aspect ratio 16:9, `ColorFilter.colorMatrix(ColorMatrix().apply { setToScale(0.7f, 0.7f, 0.7f, 1f) })` for 70% brightness darken
- Occasion pill: top-left overlay, `shapes.pill`, `colors.accent` background, `colors.paper` text, `typography.bodyXS`, `spacing.gap4` vertical + `spacing.gap8` horizontal padding
- Date: bottom-right overlay, `typography.bodyXS`, `colors.paper`, `spacing.gap8` margin from corner
- Title: `typography.displayS`, `colors.paper`, `spacing.gap8` top margin below image area
- Stats line: `typography.bodyS`, `colors.paper` at 0.6 alpha, monoCaps bullet `\u2022` separators
- Padding: `spacing.gap16` on all text content edges

**Secondary card** (all non-primary):
- Background: `colors.paperDeep`
- Border: `Modifier.border(1.dp, colors.line, shapes.radius16)`
- Image: full-brightness (no filter)
- Occasion pill: top-left overlay, `shapes.pill`, `colors.paperDeep` background, `colors.inkSoft` text
- Date: bottom-right overlay, `typography.bodyXS`, `colors.inkSoft`
- Title: `typography.displayS`, `colors.ink`
- Stats line: `typography.bodyS`, `colors.inkFaint`, monoCaps bullet `\u2022` separators

Image loading: `AsyncImage(model = registry.imageUrl, contentScale = ContentScale.Crop)`. Placeholder: `colors.paperDeep` fill. Error: `colors.paperDeep` fill.

Card tap: `Modifier.clickable { onNavigateToDetail(registry.id) }` — existing nav path preserved.

### Stats Line Construction

```kotlin
"${itemCount} items \u2022 ${reservedCount} reserved \u2022 ${givenCount} given"
```

If per-registry stats not available (Phase 10 deferral per CONTEXT.md), render "N items" where N = 0 as a known approximation. Mark with `// TODO: Phase 10 stats aggregation — see CONTEXT.md deferred section`.
