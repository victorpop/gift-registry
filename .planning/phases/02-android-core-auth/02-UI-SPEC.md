## Spacing Scale

Android dp units, 8-point scale. All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon gaps, inline chip padding |
| sm | 8dp | Compact element spacing, icon-text gap |
| md | 16dp | Default element spacing, horizontal screen padding |
| lg | 24dp | Section padding, between-form-field spacing |
| xl | 32dp | Layout gaps between major form sections |
| 2xl | 48dp | Vertical scroll offset, large section breaks |
| 3xl | 64dp | Page-level top/bottom padding |

Exceptions:
- Minimum touch target: 48dp x 48dp for all tappable elements (Material3 accessibility baseline)
- "Continue as Guest" text link: minimum 44dp vertical hit area (text-only interactive element)
- TabRow height: 48dp (Material3 default, do not reduce)

## Typography

Material3 TypeScale applied to auth screens. All text is externalized in strings.xml (no hardcoded strings per CLAUDE.md constraint).

| Role | Material3 Token | Size | Weight | Line Height |
|------|-----------------|------|--------|-------------|
| Display | headlineLarge | 32sp | Regular (400) | 1.25 |
| Heading | titleLarge | 22sp | Medium (500) | 1.27 |
| Body | bodyLarge | 16sp | Regular (400) | 1.5 |
| Label | labelLarge | 14sp | Medium (500) | 1.43 |

Notes:
- Use `headlineLarge` for the app name / screen title (auth screen top area)
- Use `titleLarge` for tab labels (Sign In / Create Account)
- Use `bodyLarge` for form input text and helper text
- Use `labelLarge` for button labels and "Continue as Guest" link text
- Do not use more than these 4 type roles in this phase

## Color

Material3 dynamic color is NOT used — define a static seed color scheme for consistent branding across all Android versions.

Seed color: `#6750A4` (Material3 default Purple — establishes brand identity before custom colors are decided)

| Role | Light Theme | Dark Theme | Usage |
|------|-------------|------------|-------|
| Dominant (60%) — `background` | `#FFFBFE` | `#1C1B1F` | Screen background, scaffold background |
| Secondary (30%) — `surface` / `surfaceVariant` | `#E7E0EC` | `#49454F` | Card background, tab bar, text field fill |
| Accent (10%) — `primary` | `#6750A4` | `#D0BCFF` | CTA buttons, selected tab indicator, text links |
| Destructive — `error` | `#B3261E` | `#F2B8B5` | Inline field errors, destructive confirmation text |

Accent (`primary`) reserved for:
1. "Sign In" filled button
2. "Create Account" filled button
3. Selected tab indicator underline
4. "Continue as Guest" text link color
5. Google Sign-In button outline stroke (use `outline` token, not `primary`)

Do NOT use `primary` for: field labels, helper text, screen titles, dividers, icon-only elements.

Source: Material3 color system defaults — no custom palette decided in CONTEXT.md (at Claude's discretion per CONTEXT.md `Claude's Discretion` section)

## Component Contract

### Auth Screen Layout

Primary focal point: app logo/wordmark at top center — largest element in the visual hierarchy (`headlineLarge` 32sp), draws the eye before the tab row and form fields.

Top-to-bottom vertical structure:

1. App logo / wordmark — centered, `headlineLarge`, top padding `3xl` (64dp)
2. `TabRow` with two tabs: "Sign In" | "Create Account" — uses `primary` for selected indicator
3. Tab content pane — animates with `AnimatedContent` on tab switch
4. **Sign In tab content:**
   - Email `OutlinedTextField` — full width, `bodyLarge`, label uses `labelLarge`
   - Password `OutlinedTextField` with visibility toggle icon — full width
   - `spacing: lg` (24dp) between fields
   - "Sign In" `Button` (filled, `primary`) — full width, `labelLarge`
   - `Spacer` with `md` (16dp)
   - "OR" divider row — `HorizontalDivider` + `Text` centered
   - `Spacer` with `md` (16dp)
   - "Sign in with Google" `OutlinedButton` — full width, Google icon prefix
5. **Create Account tab content:**
   - Email `OutlinedTextField` — full width
   - Password `OutlinedTextField` with visibility toggle — full width
   - Confirm Password `OutlinedTextField` — full width
   - `spacing: lg` (24dp) between fields
   - "Create Account" `Button` (filled, `primary`) — full width, `labelLarge`
   - `Spacer` with `md` (16dp)
   - "OR" divider row
   - `Spacer` with `md` (16dp)
   - "Sign in with Google" `OutlinedButton` — full width, Google icon prefix
6. `Spacer` with `xl` (32dp)
7. "Continue as Guest" `TextButton` — centered, `labelLarge`, `primary` color
8. Bottom padding: `lg` (24dp)

Horizontal screen padding: `md` (16dp) applied to the entire Column via `Modifier.padding(horizontal = 16.dp)`.

Source: AUTH-UX-01, AUTH-UX-02, AUTH-UX-03 decisions from CONTEXT.md

### Guest Conversion Bottom Sheet

Triggered after reservation action (Phase 4 will call into this Phase 2 component):

- `ModalBottomSheet` composable
- Heading: `titleLarge` — "Save your reservation?" (string key: `auth_guest_conversion_title`)
- Body: `bodyLarge` — "Create a free account to track your reservation and receive updates." (string key: `auth_guest_conversion_body`)
- Primary action: "Create Account" filled `Button` — full width, `primary`
- Secondary action: "Maybe later" `TextButton` — full width, dismisses sheet
- Sheet height: wrap content, min 200dp
- Dismissible by swipe down

Source: AUTH-UX-04 decision from CONTEXT.md

### Settings Screen Layout

1. `TopAppBar` with back navigation — title "Settings" (`auth_settings_title` key)
2. `ListItem` for language selection — leading icon: language icon, headline: "Language", supporting text: current language name
3. Language picker: `AlertDialog` with `RadioButton` list — "English" / "Romana"
4. Confirmation via "Change Language" button (`settings_language_confirm`) — triggers `Activity.recreate()` to apply locale

Source: I18N-UX-02, I18N-UX-03 decisions from CONTEXT.md

## Interaction States

All interactive elements must implement these states:

| Element | States Required |
|---------|----------------|
| Text fields | Default, focused (primary outline), error (error outline + helper text), disabled |
| Buttons (filled) | Default, pressed (ripple), loading (CircularProgressIndicator replaces label), disabled |
| Buttons (outlined/text) | Default, pressed (ripple), disabled |
| Tabs | Selected (primary indicator), unselected |
| Bottom sheet | Open, dismissed |

Loading state: replace button label with `CircularProgressIndicator(modifier = Modifier.size(24.dp))` — do not disable button separately, the loading state implies non-interactability.

## Accessibility Contract

Minimum requirements for this phase (Material3 baseline):

- All `IconButton` elements must have `contentDescription` set — use string resources
- Password field visibility toggle: `contentDescription` must update based on current state (`auth_password_show` / `auth_password_hide`)
- "Continue as Guest" `TextButton`: minimum 44dp vertical hit area (use `Modifier.defaultMinSize(minHeight = 44.dp)`)
- Color contrast: Material3 default token values meet WCAG AA — do not override with lower-contrast values
- `semantics { heading() }` on screen title text
