# Phase 9 — UI Design Contract

> Visual and interaction contract for Phase 9: Shared Chrome + Status UI.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope reminder:** This phase ships reusable cross-cutting composables only —
> bottom nav, centre FAB, Add-action bottom sheet, status chips, purchased-row
> modifier. The owner screens themselves (Home, RegistryDetail, Settings,
> StoreList) keep their current layouts until Phases 10–11 consume these
> components. RegistryDetailScreen is re-pointed to the new shared status chips
> as part of this phase.

The following `GiftMaisonSpacing` tokens are consumed by Phase 9 components. The executor must verify each token exists in the shipped Phase 8 `GiftMaisonSpacing.kt` before use. If any token is missing, add it to the Phase 8 spacing scale or escalate before proceeding.

Exceptions (hardcoded pixel values — not spacing tokens, justified by handoff):
- Bottom nav vertical padding: 4 dp top / 4 dp mid / 6 dp bottom (handoff "4/4/6 padding") — use `gap4` for top/mid, `gap6` for bottom.
- Centre FAB lift: 22 dp above the nav bar baseline — absolute offset value, not a spacing token. Hard-code as `offset(y = (-22).dp)` at the call site and comment the source.

- Drag handle: 36 dp × 4 dp — hardcoded per handoff.
- Touch target floor: 44 dp minimum on all interactive elements (accessibility). Nav items, FAB, action sheet rows, and status chip overflow buttons must all satisfy this.

## Typography

> **All size, weight, and line-height values are fixed in Phase 8 `GiftMaisonTypography.kt` and re-used unchanged. Phase 9 does not declare or override any typography scale values — this table only documents which existing Phase 8 roles each Phase 9 component consumes.**

Phase 9 consumes the Phase 8 type scale exclusively. No new sizes or weights are introduced.

| Role | GiftMaisonTypography field | Phase 9 Usage |
|------|---------------------------|---------------|
| Mono caps | `monoCaps` | Nav tab labels; "ADD" caption under FAB; status chip text (Reserved/Given/Open) |
| Body M emphasis | `bodyMEmphasis` | Action sheet row primary label (e.g. "New registry") |
| Body XS | `bodyXS` | Action sheet row subtitle (e.g. "and invite friends to add gifts") |
| Display M | `displayM` | Add-action sheet title: "What are you adding?" |
| Body S | `bodyS` | Countdown "Nm" text inside Reserved chip |

Typography rules:
- Nav tab labels: `monoCaps` (JetBrains Mono, all-caps via string resource). Colour: unselected = `colors.inkFaint`; selected = `colors.accent`. Selected-state weight override: see `GiftMaisonBottomNav` component entry below.
- "ADD" caption under FAB slot: `monoCaps`, `colors.inkFaint`, centred below the FAB at the bar baseline.
- Countdown label: `bodyS`, `colors.accentInk` (on filled accent chip). Format: `"%dm"` with trailing lowercase "m" — string resource `status_chip_countdown_template` = `"%dm"` (en) / `"%d min"` (ro).
- Action sheet title: `displayM` (Instrument Serif italic style — use `FontStyle.Italic` on the composable). Colour: `colors.ink`.
- All composables must not hardcode `includeFontPadding` — Phase 8 `noFontPadding` platform style is inherited through the `TextStyle` objects.

Anatomy:

- Row height: minimum 56 dp (touch target).

- Heading: `bodyMEmphasis`, `colors.ink`.
- Subtitle: `bodyXS`, `colors.inkSoft`.
- Trailing chevron: `Icons.AutoMirrored.Filled.ChevronRight`, `colors.inkFaint`, 18 dp.

Action rows (in order):
| Row | Icon glyph | Heading string resource | Subtitle |
|-----|-----------|------------------------|---------|

| Item from URL | `⌕` (or Material `Link`) | `add_sheet_item_url` | `add_sheet_item_url_sub` |

Zero-registry guard: "Item from URL", "Browse stores", and "Add manually" rows show a registry picker sub-sheet when tapped from Home (no registry pre-selected). If owner has zero registries, the rows are visually disabled (`alpha = 0.38f`) and tapping shows an inline helper text below the row: `add_sheet_no_registry_hint` ("Create a registry first").

#### `ui/common/status/StatusChip.kt`

```
ReservedChip(expiresAt: Instant?, modifier: Modifier = Modifier)
GivenChip(modifier: Modifier = Modifier)
OpenChip(modifier: Modifier = Modifier)
StatusChip(status: ItemStatus, expiresAt: Instant?, modifier: Modifier = Modifier)
```

`StatusChip` is the top-level dispatcher — routes to the correct chip based on `ItemStatus` enum.

**ReservedChip anatomy:**
- Pill shape (`shapes.pill`), `colors.accent` fill.
- Row contents: `PulsingDot` (4 dp, `colors.accentInk`, 1400 ms period) + `gap4` + countdown label + `gap8` + chip label.
- Countdown label: `bodyS`, `colors.accentInk`. Derives `minutesLeft = max(0, (expiresAt - now).inWholeMinutes)`. Format: `stringResource(R.string.status_chip_countdown_template, minutesLeft)` → `"23m"`. When `minutesLeft == 0`: show `"<1m"` (string resource `status_chip_countdown_zero`).
- Chip label: `monoCaps`, `colors.accentInk`, text = `stringResource(R.string.status_chip_reserved)` → `"RESERVED"`.
- Countdown is display-only on owner screens. No tap handler. `clickable = false`.
- Countdown re-render: `LaunchedEffect(expiresAt)` loop with `delay(60_000L)` — no ViewModel changes, no shared StateFlow.
- When `expiresAt` is null (legacy items predating the field): omit the countdown label; show chip label only.
- Chip padding: 6 dp horizontal, 4 dp vertical.

**GivenChip anatomy:**
- Pill shape, `colors.secondSoft` fill.
- Label: `monoCaps`, `colors.second`, text = `stringResource(R.string.status_chip_given)` → `"✓ GIVEN"`.
- Padding: same as ReservedChip.

**OpenChip anatomy:**
- Pill shape, transparent background, `1.dp` border `colors.line`.
- Label: `monoCaps`, `colors.inkFaint`, text = `stringResource(R.string.status_chip_open)` → `"OPEN"`.
- Padding: same as ReservedChip.

#### `ui/common/status/PulsingDot.kt`

```
PulsingDot(
    color: Color,
    size: Dp = 4.dp,
    period: Duration = 1_400.milliseconds,
    modifier: Modifier = Modifier,
)
```

Implementation: `rememberInfiniteTransition(label = "pulsingDot")` animating:
- Alpha: `1f` ↔ `0.5f` via `tween(durationMillis = period.inWholeMilliseconds.toInt() / 2, easing = FastOutSlowInEasing)`, `RepeatMode.Reverse`.
- Scale: `1f` ↔ `0.85f` via same tween + repeat.

Result: `Box(modifier = Modifier.size(size).graphicsLayer { scaleX = scale; scaleY = scale; alpha = alpha }.background(color, CircleShape))`.

Used by:
- `ReservedChip`: 4 dp, `colors.accentInk`, 1400 ms.
- Phase 11 "Fetching from…" URL field: 4 dp, `colors.accent`, 1000 ms. Same composable, different params.

#### `ui/common/status/PurchasedRowModifier.kt`

```kotlin
fun Modifier.purchasedVisualTreatment(): Modifier
```

Applies:
1. `Modifier.alpha(0.55f)` on the whole row.
2. Image treatment: caller applies `colorFilter = ColorFilter.colorMatrix(ColorMatrix().apply { setToSaturation(0f) })` on the `AsyncImage` composable, plus a `Box` overlay of `colors.ink.copy(alpha = 0.4f)` fill and a centred `Icon(Icons.Default.Check, tint = colors.paper, size = 20.dp)`.
3. Title treatment: caller applies `TextDecoration.LineThrough` to the title `Text`.

Note: `purchasedVisualTreatment()` applies the row-level alpha. Image colorFilter and title decoration are applied at the individual element level by the caller (cannot be done via a Modifier alone in Compose). Document this in the KDoc so Phase 11 executors know the split responsibility.

## Interaction Contracts

### Bottom nav tap behaviour

| Tap target | Current key is HomeKey | Current key is RegistryDetailKey |
|-----------|------------------------|----------------------------------|
| Home | No-op | Pop to HomeKey (`backStack.clear()`, add HomeKey) |
| Stores | Push StoreListKey | Push StoreListKey |
| FAB | Open AddActionSheet | Open AddActionSheet |
| Lists | Push RegistryDetailKey(isPrimaryId) | No-op (already on detail) |
| You | Push SettingsKey | Push SettingsKey |

"Lists" tab `isPrimaryId` resolver (Phase 9 version): first registry from `RegistryListViewModel.registries.value` ordered by `updatedAt desc`. If empty, navigate to an empty-state composable that renders the `add_sheet_new_registry` suggestion (inline within the nav transition — not a crash, not a silent no-op). Phase 10 refines this with the real `isPrimary` field.

### FAB Add-action sheet dismissal

- Scrim tap → dismiss (ModalBottomSheet default behaviour).
- Swipe down → dismiss (ModalBottomSheet default behaviour).
- Back gesture → dismiss before popping backstack.

### Status chip interactions

All status chips are non-interactive (no tap handler) in Phase 9. The overflow `⋯` button on RegistryDetailScreen item rows remains on the screen and retains its existing tap behaviour (edit/delete menu) — it is not part of Phase 9's new components.

## Implementation Notes for Executor

1. **Single integration point** — `AppNavigation.kt` (373 lines). The `Scaffold(bottomBar = { ... })` block is the only place that changes. Replace the current `NavigationBar { NavigationBarItem(...) x4 }` with `GiftMaisonBottomNav(...)`. Delete the current `showsBottomNav()` function and replace with the new key-based predicate.
