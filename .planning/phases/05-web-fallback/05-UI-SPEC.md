# Phase 5 — UI Design Contract

> Visual and interaction contract for the web fallback. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This phase is giver-only. No owner features exist on the web.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind CSS + Radix UI — WEB-D-03) |
| Preset | not applicable |
| Component library | Radix UI Primitives (`@radix-ui/react-dialog`, `@radix-ui/react-toast`, `@radix-ui/react-alert-dialog`, `@radix-ui/react-label`, `@radix-ui/react-visually-hidden`) |
| Icon library | Lucide React (`lucide-react`) |
| Font | Inter (Google Fonts CDN; subset: latin + latin-ext for Romanian diacritics) |

**Radix UI rationale (Claude's Discretion — WEB-D-03):** Radix is chosen over Headless UI and ark-ui for this phase because (1) it ships unstyled primitives with zero runtime CSS injection — compatible with Tailwind's atomic class approach; (2) it has first-class ARIA implementations for Dialog, Toast, and AlertDialog, which are all required in this phase; (3) bundle cost is per-primitive (no monolithic install); (4) Headless UI 2.x still requires Floating UI for positioning and has looser TypeScript types; (5) ark-ui is Zag.js-based with more abstractions than this phase needs.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Tailwind Class | Value | Usage |
|-------|---------------|-------|-------|
| xs | `p-1` / `gap-1` | 4px | Icon gaps, inline badge padding |
| sm | `p-2` / `gap-2` | 8px | Compact element spacing, icon-to-label gaps |
| md | `p-4` / `gap-4` | 16px | Default element spacing, form field vertical gaps |
| lg | `p-6` / `gap-6` | 24px | Card inner padding, section padding |
| xl | `p-8` / `gap-8` | 32px | Layout column gaps |
| 2xl | `p-12` / `gap-12` | 48px | Major section breaks |
| 3xl | `p-16` / `gap-16` | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets (mobile): minimum 44px height on all interactive elements via `min-h-[44px]`
- Reserve button: minimum 48px height (`min-h-[48px]`) — primary CTA must be finger-friendly on mobile browsers
- Guest identity modal inputs: 48px height (`h-12`) for comfortable mobile keyboard interaction

## Typography

Font: Inter loaded via Google Fonts (`weights=400;600`, `subsets=latin,latin-ext`).

| Role | Size | Tailwind | Weight | Tailwind | Line Height |
|------|------|----------|--------|----------|-------------|
| Body | 16px | `text-base` | 400 (Regular) | `font-normal` | 1.5 (`leading-relaxed`) |
| Label | 14px | `text-sm` | 400 (Regular) | `font-normal` | 1.4 (`leading-snug`) |
| Heading | 20px | `text-xl` | 600 (Semibold) | `font-semibold` | 1.2 (`leading-tight`) |
| Display | 28px | `text-[28px]` | 600 (Semibold) | `font-semibold` | 1.2 (`leading-tight`) |

Only 2 weights in use: 400 and 600. Do not introduce 500 or 700.

Display (28px) used only for: registry name at the top of `/registry/:id`. All other section titles use Heading (20px).

## Color

Color tokens mirror the Android Material3 light theme (source: `app/src/main/java/com/giftregistry/ui/theme/Color.kt`, Phase 2 decision: static seed `#6750A4`). Dark mode is out of scope for Phase 5 web fallback.

| Role | Hex | Tailwind Config Key | Usage |
|------|-----|---------------------|-------|
| Dominant (60%) | `#FFFBFE` | `colors.surface.DEFAULT` | Page background, all surface backgrounds |
| Secondary (30%) | `#E7E0EC` | `colors.surface.variant` | Item cards, modal backdrop, nav bar background |
| Accent (10%) | `#6750A4` | `colors.primary.DEFAULT` | Reserve button, countdown badge, active link underline |
| On-Accent | `#FFFFFF` | `colors.primary.on` | Text on accent-colored elements |
| On-Surface | `#1C1B1F` | `colors.surface.on` | Primary body text |
| On-Surface-Variant | `#49454F` | `colors.surface.onVariant` | Secondary body text, placeholder text |
| Destructive | `#B3261E` | `colors.destructive.DEFAULT` | "Release reservation" destructive action only |
| On-Destructive | `#FFFFFF` | `colors.destructive.on` | Text on destructive elements |
| Border | `#CAC4D0` | `colors.outline.DEFAULT` | Input borders, card borders |
| Skeleton | `#E7E0EC` | (reuse surface.variant) | Skeleton loading shimmer base |

Accent (`#6750A4`) is reserved for ONLY these elements:
1. Reserve button (primary CTA) background
2. Countdown badge background on reserved item cards
3. Sticky "Your reservation" banner background
4. Underline on the active nav/breadcrumb link
5. Focus rings (`ring-2 ring-primary`) on all interactive elements — treated as a system-wide a11y affordance, not decorative accent usage. Does not count toward the 10% visual weight budget.

No other element may use the accent color. Available item status badges use `#1C1B1F` on `#E7E0EC` background (surface-variant), not accent.

**Tailwind config extension required:**
```js
// tailwind.config.ts extend.colors block
colors: {
  primary:  { DEFAULT: '#6750A4', on: '#FFFFFF' },
  surface:  { DEFAULT: '#FFFBFE', variant: '#E7E0EC', on: '#1C1B1F', onVariant: '#49454F' },
  destructive: { DEFAULT: '#B3261E', on: '#FFFFFF' },
  outline:  { DEFAULT: '#CAC4D0' },
}
```

## Component Inventory

Components to build in Phase 5, with their Radix primitive and visual spec:

### ItemCard
- Radix primitive: none (plain `<article>`)
- Layout: image left (96x96px, `object-cover rounded-md`), text block right, status badge top-right
- Status badge states:
  - Available: text `available` key, background `surface.variant`, text `surface.on`, no accent
  - Reserved: text `reservation.status_reserved` key, background `primary.DEFAULT`, text `primary.on`
  - Purchased: text `reservation.status_purchased` key, background `surface.on`, text `surface.DEFAULT`
- Countdown on reserved items: inline below badge, text `text-sm font-normal`, same `primary.DEFAULT` background as badge
- Reserve button: hidden when status is reserved or purchased; shown when available

### ReserveButton
- Element: `<button>` with `min-h-[48px] px-6 rounded-full bg-primary text-primary-on font-semibold text-base`
- Loading state: button disabled, inline spinner (16px) replaces label text; use `aria-busy="true"`
- Text: `reservation.reserve_item` i18n key (EN: "Reserve Gift", RO: "Rezervă Cadoul")

### GuestIdentityModal
- Radix primitive: `@radix-ui/react-dialog`
- Triggered on Reserve click when user is not authenticated
- Fields: First name, Last name, Email — all required
- Field height: 48px (`h-12`), border `outline.DEFAULT`, focus ring `ring-2 ring-primary`
- CTA inside modal: same ReserveButton spec
- Pre-fill from `localStorage.guestIdentity` if present
- Dismissal: `Escape` key + backdrop click closes modal without submitting

### AuthModal
- Radix primitive: `@radix-ui/react-dialog`
- Triggered from "Sign in" header link
- Tabs: "Sign In" / "Create Account" — tab switch via toggle buttons (no Radix Tabs needed at this scale)
- Google OAuth: button with Google logo SVG (inline, not icon library), text "Continue with Google"
- Separator line: "or" text with `hr` lines either side
- Below the form: "Continue as guest" link (text-sm, underline on hover) that dismisses the modal

### ReservationBanner (sticky)
- Fixed at top of page below nav, full-width, `bg-primary text-primary-on`
- Shown when: current visitor has an active reservation in localStorage state
- Content: "You reserved [item name] — [countdown] left" + "View on [retailer]" anchor `target=_blank`
- Height: 48px, text `text-sm font-semibold`
- Dismissed automatically when countdown reaches 0; no manual dismiss

### SkeletonCard
- Visually matches ItemCard layout; elements replaced with `animate-pulse` divs
- Image placeholder: 96x96 `rounded-md bg-surface-variant`
- Title placeholder: `h-4 w-3/4 rounded bg-surface-variant`
- Price placeholder: `h-4 w-1/4 rounded bg-surface-variant`
- Show 6 skeleton cards on initial load before data arrives

### RegistryHeader (skeleton-capable)
- Registry name: Display (28px semibold)
- Occasion + date: Body (16px regular), `surface.onVariant` color
- Skeleton: two rows, `animate-pulse bg-surface-variant rounded`

### NotFoundPage
- Generic "Registry not available" — used for deleted, bad ID, and permission-denied (WEB-D-13/14)
- Heading: Heading (20px semibold): i18n key `registry.not_found_title`
- Body: Body (16px): i18n key `registry.not_found_body`
- No link back to home (there is no home page for givers)

### Toast (reservation feedback)
- Radix primitive: `@radix-ui/react-toast`
- Success: "Gift reserved! Redirecting to retailer…" — `surface.DEFAULT` bg, `surface.on` text, `primary` left border 4px
- Error ITEM_UNAVAILABLE: "Someone just reserved this gift. Try another." — `destructive.DEFAULT` left border 4px
- Error generic: `common.error_generic` i18n key
- Duration: 5000ms, position: bottom-center

## Page Layout

### `/registry/:id` — Registry Detail Page

```
┌──────────────────────────────────────┐
│ [Nav: app name | Sign in link]       │  48px, bg-surface-variant, sticky
│ [ReservationBanner — conditional]   │  48px, bg-primary, sticky below nav
├──────────────────────────────────────┤
│ [RegistryHeader]                     │  pt-16 pb-8, max-w-2xl mx-auto px-4
│  Registry name (Display 28px)        │
│  Occasion, date, location (Body 16px)│
├──────────────────────────────────────┤
│ [Item grid — 1 col mobile, 2 col md] │  gap-4, max-w-2xl mx-auto px-4 pb-16
│  [ItemCard]                          │
│  [ItemCard]                          │
│  ...                                 │
└──────────────────────────────────────┘
```

- Max content width: `max-w-2xl` (672px) centered — sufficient for item cards, avoids wide layout complexity
- Mobile: single column (`grid-cols-1`)
- Desktop (≥768px): two columns (`md:grid-cols-2`)
- No sidebar. No pagination. Infinite scroll is out of scope — items render in a single list.
- **Primary visual anchor:** the `ReserveButton` on each `ItemCard` — it is the only accent-colored interactive element visible at rest. Secondary anchor: the registry name (Display 28px semibold) in `RegistryHeader`. When `ReservationBanner` is active, it becomes the topmost accent surface and claims primary anchor until dismissed.

### `/reservation/:id/re-reserve` — Re-Reserve Deep Link

No visible page UI. On mount: calls `resolveReservation` callable, then redirects to `/registry/:id?autoReserveItemId=:itemId`. Shows a centered spinner + "Checking your reservation…" text while the callable is in flight. On error: redirects to `/` (home fallback) — no registry to show.

### `/` (root fallback)

Out of scope for this phase — no home page for givers. Root redirects to a minimal "Get the app" static page. Visual spec: plain page, app name (Display 28px), subtitle "Download for Android" (Body 16px), no Tailwind design needed beyond centering.

## Accessibility Contract

- All Radix Dialog/Toast primitives include ARIA roles automatically
- Images: all `<img>` elements require `alt` text — item images use item title as alt; skeleton images use `aria-hidden="true"`
- Color contrast: `#6750A4` on `#FFFFFF` = 4.55:1 (AA for normal text); `#1C1B1F` on `#FFFBFE` = 17.3:1 (AAA)
- Focus management: modal open moves focus to first focusable element; modal close returns focus to trigger
- Reserve button loading state: `aria-busy="true"` + `aria-label` updated to "Reserving gift…" during loading
