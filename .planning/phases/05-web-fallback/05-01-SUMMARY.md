## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing critical config] Added `passWithNoTests: true` to vitest config**
- **Found during:** Task 2 verification (`npm run test:run`)
- **Issue:** vitest 2.x exits with code 1 when no test files are found. Plan requires empty suite to pass green.
- **Fix:** Added `passWithNoTests: true` to `test` block in vitest.config.ts
- **Files modified:** web/vitest.config.ts
- **Commit:** 6d8b93d

**2. [Rule 1 - Bug] Used `react() as any` in vitest.config.ts to resolve type mismatch**
- **Found during:** Task 2 build verification (`npm run build` → `tsc -b`)
- **Issue:** vitest 2.x bundles its own internal vite copy; the `@vitejs/plugin-react` Plugin type from the outer `vite@6` package is incompatible with vitest's bundled vite Plugin type at the TypeScript level. Build failed with TS2769.
- **Fix:** Used `import { defineConfig } from 'vitest/config'` (which adds `test` property augmentation) and `react() as any` for the plugin type cast
- **Files modified:** web/vitest.config.ts
- **Commit:** 6d8b93d

**3. [Rule 2 - Missing critical config] Migrated assetlinks.json to web/public/**
- **Found during:** Task 2 build verification
- **Issue:** `emptyOutDir: true` deletes the entire `hosting/public/` directory before each build, including the previously-committed `hosting/public/.well-known/assetlinks.json` (Android App Links, set up Phase 1). This file would be lost on every build.
- **Fix:** Moved `assetlinks.json` from `hosting/public/.well-known/` to `web/public/.well-known/`. Vite copies all files from `web/public/` to the output directory on each build, so the file survives the emptyOutDir cycle.
- **Files modified:** .gitignore, web/public/.well-known/assetlinks.json (new location)
- **Commit:** 6d8b93d

**4. [Rule 2 - Missing critical config] Updated .gitignore to exclude Vite build output**
- **Found during:** Task 2 post-build git status check
- **Issue:** `hosting/public/assets/` and other Vite build artifacts were untracked (not gitignored). These are generated files that should not be committed.
- **Fix:** Added `hosting/public/assets/`, `hosting/public/index.html`, `hosting/public/vite.svg`, `hosting/public/.well-known/` to root `.gitignore`. Also added `web/node_modules/`, `web/package-lock.json`, `web/tsconfig.*.tsbuildinfo` to gitignore.
- **Files modified:** .gitignore
- **Commit:** 6d8b93d
