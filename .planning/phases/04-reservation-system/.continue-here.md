<completed_work>

**App launch crashes (Phase 2 regressions):**
- Fix 1: `lateinit languagePrefsRepo` accessed before `super.onCreate()` ŌĆö moved locale restore to `GiftRegistryApp.onCreate()`
- Fix 2: `NavigationEventDispatcherOwner` not set ŌĆö AppCompat 1.7.0 skips it in `initViewTreeOwners()`. Manually set on decor view in `MainActivity`

**Navigation3 / SavedStateHandle gap (Phase 3):**
- Fix 3: `RegistryDetailViewModel` crashed on `checkNotNull(savedStateHandle["registryId"])` ŌĆö Nav3 doesn't populate SavedStateHandle from nav keys like Nav2 did
- Fix 4: Swept `AddItemViewModel`, `EditItemViewModel`, `CreateRegistryViewModel` for same bug
- Fix 5: Created shared `hiltViewModelWithNavArgs()` helper (`HiltViewModelNavArgs.kt`) that seeds `CreationExtras[DEFAULT_ARGS_KEY]` with a bundle, builds Hilt factory explicitly, and goes through `viewModel()` (hilt-navigation-compose 1.2.0 lacks `extras` param on `hiltViewModel()`)

**Cloud Functions region + admin v13 import issues (Phase 3-4):**
- Fix 6: `fetchOgMetadata` returned NOT_FOUND ŌĆö `FirebaseFunctions.getInstance()` defaulted to `us-central1`, all functions are `europe-west3`. Fixed in `AppModule.kt`
- Fix 7: Added currency handling to `fetchOgMetadata` ŌĆö JSON-LD walker, itemprop, OG tags, inline string parsing with `CURRENCY_ALIASES` normalization. New response fields `priceAmount` + `priceCurrency`
- Fix 8: Invite crash ŌĆö dotted email keys (e.g. `email:jane.doe@example.com`) in `update({"invitedUsers.key": true})` created nested maps. Fixed with `FieldPath` constructor
- Fix 9: `admin.firestore.FieldPath is not a constructor` ŌĆö namespace access broken in firebase-admin v13 ESM interop. Fixed with `import { FieldPath } from "firebase-admin/firestore"` subpath
- Fix 10: Swept `createReservation.ts` and `releaseReservation.ts` for same `admin.firestore.Timestamp` / `FieldValue` issue ŌĆö all fixed to subpath imports

**Deserialization hardening (Phase 3):**
- Fix 11: `RegistryDto.invitedUsers` widened to `Map<String, Any?>` with domain-layer coercion to `Boolean`. Prevents one malformed doc from crashing the snapshot listener

**Auth UI gap (Phase 2):**
- Fix 12: "Create Account" button was a TODO no-op. Wired sign-in/sign-up mode toggle with confirm password field, mode-aware button label/action, and "Already have an account?" / "Don't have an account?" toggle text
