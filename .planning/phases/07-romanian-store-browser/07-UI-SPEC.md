## Spacing Scale

Android dp units, 8-point scale. Unchanged from Phases 2–6. Do not override.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon gaps, inline chip padding |
| sm | 8dp | Compact element spacing, icon-text gap, card grid gap |
| md | 16dp | Default element spacing, horizontal screen padding |
| lg | 24dp | Section padding |
| xl | 32dp | Layout gaps between major sections |
| 2xl | 48dp | Vertical scroll offset, large section breaks |
| 3xl | 64dp | Store logo size, page-level spacing |

Exceptions for this phase:
- Store card minimum tap target: 100dp height (card height, not just the tap area — satisfies 48dp touch target with headroom for logo + label)
- Store logo inside card: 64dp × 64dp, centered horizontally
- Bottom "Add to list" bar: 56dp button height per Material3 primary button spec (not 48dp — this is a full-width primary action bar, not a list row)
- Bottom bar total height: 72dp (56dp button + 8dp padding top + 8dp padding bottom); accounts for navigation bar insets via `WindowInsets.navigationBars` padding applied to the Scaffold bottomBar

Source: CONTEXT.md `<specifics>` + Material3 button height spec

## Color

Material3 color scheme from seed `#6750A4` — unchanged from Phase 2. Apply `MaterialTheme.colorScheme.*` tokens throughout. No new colors introduced.

| Role | Light value | Dark value | Usage |
|------|------------|-----------|-------|
| Dominant (60%) | #FFFBFE | #1C1B1F | Scaffold background, screen background, WebView Scaffold |
| Secondary (30%) | #E7E0EC (surfaceVariant) | #49454F | Store cards (`Card` container), bottom CTA bar surface |
| Accent (10%) | #6750A4 (primary) | #D0BCFF | Reserved for: "Add to list" filled primary button, FAB menu items' icon tint (active), close button if using `OutlinedButton` variant |
| Destructive | #B3261E (error) | #F2B8B5 | Not used in this phase |

Accent (`primary`) reserved specifically for:
1. "Add to list" `Button` (filled) in WebView bottom bar — the single primary CTA
2. `FloatingActionButton` and its extended variant on the Home screen (existing use, unchanged)
3. FAB menu action items: use `FilledTonalButton` (secondary container) for "Browse stores" and "Create registry" menu options — accent reserved for the FAB itself, not the expanded menu items

Bottom CTA bar surface: use `MaterialTheme.colorScheme.surface` with `Modifier.shadow(elevation = 8.dp)` to lift the bar above page content. Do not use `surfaceVariant` for the bar background — it must read as a distinct action zone above the WebView.

Store card container color: `MaterialTheme.colorScheme.surface` with `CardDefaults.cardElevation(defaultElevation = 2.dp)` — matches `RegistryCard` pattern (2dp elevation, confirmed via `RegistryListScreen.kt`).

Source: Phase 2 UI-SPEC.md, Phase 6 UI-SPEC.md, `Color.kt` confirmed existing

## Component Contracts

### 1. Home Screen FAB Menu (refactor of existing `RegistryListScreen` FAB)

**Current state:** Single `ExtendedFloatingActionButton` with `Icons.Default.Add` and label from `R.string.registry_create_button`.

**New pattern:** FAB expands to reveal a small action menu on tap. Use the existing M3 `ExtendedFloatingActionButton` as the toggle — do NOT use a speed-dial pattern (requires third-party library). Instead: toggle a `DropdownMenu` anchored to the FAB position, or use an `AnimatedVisibility` column of two `SmallFloatingActionButton` items above the main FAB (M3 FAB menu pattern without library dependency).

**Chosen pattern:** Vertical stack of `SmallFloatingActionButton` items above the main FAB, revealed with `AnimatedVisibility(visible = menuExpanded)` using `expandVertically() + fadeIn()` exit/enter. This uses only M3 standard components.

**Layout (bottom-up):**
1. Main `ExtendedFloatingActionButton` — `Icons.Default.Add` icon, label from `stores_fab_label` key (`"New"` / `"Nou"`)
   - On tap when collapsed: expand menu (`menuExpanded = true`)
   - On tap when expanded: collapse menu (`menuExpanded = false`)
   - Icon animates from `Add` → `Close` (using `AnimatedContent` on icon slot, rotate 45° or swap icon)
2. `AnimatedVisibility` column (appears above FAB when expanded):
   - `Row` with `SmallFloatingActionButton` + `Text` label (M3 "FAB menu item" pattern):
     - Item 1: `Icons.Default.Storefront` (or `Icons.Default.ShoppingBag` as fallback if `Storefront` unavailable) — label `"Browse stores"` / `"Răsfoiește magazine"` (`stores_browse_label`)
     - Item 2: `Icons.Default.Edit` — label `"Create registry"` / `"Creează o listă"` (`stores_create_registry_label`) — existing action
   - Each item: `SmallFloatingActionButton` (40dp) + `Text` (labelMedium, 12sp) to the left, right-aligned as a row

**Touch target:** Both `SmallFloatingActionButton` items have minimum 48dp touch target (M3 default for SmallFAB is 40dp visible, 48dp touch area via padding).

**Scrim:** `Box` behind the FAB menu column uses `Modifier.clickable { menuExpanded = false }` when `menuExpanded = true` — allows tapping outside to dismiss. No overlay color change needed (menu is small).

Source: CONTEXT.md D-11, `<specifics>` section; `RegistryListScreen.kt` confirms current FAB is `ExtendedFloatingActionButton`

### 2. Store List Screen

**Route:** `StoreListKey`

**Scaffold structure:**
- `TopAppBar`: title = `"Browse stores"` / `"Răsfoiește magazine"` (`stores_list_title`), back navigation arrow (`Icons.AutoMirrored.Filled.ArrowBack`)
- Content: `LazyVerticalGrid(columns = GridCells.Fixed(2), contentPadding = PaddingValues(16.dp), horizontalArrangement = Arrangement.spacedBy(8.dp), verticalArrangement = Arrangement.spacedBy(8.dp))`

**Store Card layout (each cell in grid):**
- `Card(modifier = Modifier.fillMaxWidth().height(100.dp), elevation = CardDefaults.cardElevation(defaultElevation = 2.dp))`
- `Column(horizontalAlignment = Alignment.CenterHorizontally, verticalArrangement = Arrangement.Center, modifier = Modifier.fillMaxSize().padding(8.dp))`
  - Logo: `Image(painter = painterResource(logoResId), modifier = Modifier.size(64.dp))` — resolved via `context.resources.getIdentifier(logoAsset, "drawable", packageName)`, fallback to `R.drawable.store_generic`
  - `Spacer(4.dp)`
  - Store name: `Text(store.name, style = MaterialTheme.typography.labelLarge, textAlign = TextAlign.Center, maxLines = 2, overflow = TextOverflow.Ellipsis)`
- Card `onClick`: navigate to `StoreBrowserKey(storeId, registryId)` — `registryId` is non-null only when entered from Registry Detail "Browse stores" shortcut
- No long-press action, no context menu on store cards

**States:**

| State | Trigger | Visual |
|-------|---------|--------|
| Loading | Firestore fetch in progress | `CircularProgressIndicator` centered in content area |
| Success | `stores` list non-empty | `LazyVerticalGrid` of store cards |
| Empty / Error | Firestore doc missing or fetch failure | Error state: centered `Icons.Default.CloudOff` (48dp, `onSurfaceVariant` tint) + `titleMedium` heading + `bodyLarge` body + `OutlinedButton` retry |


**Primary visual anchor:** The store card grid (64dp logo + store name label pattern) is the focal point of this screen. Cards should dominate the viewport; the TopAppBar is secondary chrome. Ensure logos are loaded promptly and the 2-column grid fills the screen width with minimal whitespace.

Source: CONTEXT.md D-17, `<specifics>` section

### 3. Store Browser Screen

**Route:** `StoreBrowserKey(storeId: String, registryId: String?)`

**Scaffold structure:**
```
Scaffold(
  topBar   = TopAppBar,
  content  = AndroidView(::WebView),   // fills weight(1f) between topBar and bottomBar
  bottomBar = AddToListBar
)
```

**TopAppBar:**
- Title: store name (from `StoreBrowserViewModel` loaded from `storeId`) — `titleLarge` via M3 default
- Navigation icon: `Icons.Default.Close` with `contentDescription = stringResource(stores_webview_close)` — navigates back to Store List (pops back stack)
- No actions in toolbar

**WebView (`AndroidView`):**
- Fills remaining scaffold content area between top and bottom bars
- `webSettings.javaScriptEnabled = true` (D-05)
- `webSettings.domStorageEnabled = true` (required by most Romanian e-commerce sites)
- `CookieManager.getInstance().setAcceptCookie(true)` (D-06)
- `WebViewClient.onPageFinished(view, url)`: pushes `url` to `StoreBrowserViewModel.currentUrl: StateFlow<String>` (D-07)
- `WebViewClient.onReceivedError(...)`: sets `StoreBrowserViewModel.pageLoadFailed = true`, pushes empty string to `currentUrl` (D-15)
- `WebViewClient.shouldOverrideUrlLoading(...)`: return `true` for non-http/https schemes; show `Snackbar` with `stores_external_link_blocked` message (D-08)
- Load initial URL from store's `homepageUrl` on composable entry

**Error state (overlaid on WebView area):**
Shown when `pageLoadFailed == true`. Rendered as a `Box` overlaying the WebView area (WebView hidden via `Modifier.alpha(0f)` not removed from composition — preserves WebView instance).

Layout (centered column):
- Icon: `Icons.Default.WifiOff` (48dp, `onSurfaceVariant` tint)
- `Spacer(16.dp)`
- Heading: `stringResource(stores_webview_error_heading)` — `titleMedium`, center-aligned
- `Spacer(8.dp)`
- Body: `stringResource(stores_webview_error_body)` — `bodyLarge`, `onSurfaceVariant` color, center-aligned
- `Spacer(24.dp)`
- `Button` (filled, primary): `stringResource(stores_retry)` — reloads `webView.reload()`, resets `pageLoadFailed = false`
- `Spacer(8.dp)`
- `OutlinedButton`: `stringResource(common_back)` — pops back stack

**Bottom "Add to list" bar:**
- Surface: `Surface(tonalElevation = 8.dp, modifier = Modifier.fillMaxWidth())`
- Padding: `Modifier.padding(horizontal = 16.dp, vertical = 8.dp).windowInsetsPadding(WindowInsets.navigationBars)`
- Single child: `Button(modifier = Modifier.fillMaxWidth().height(56.dp), enabled = !pageLoadFailed && currentUrl.isNotBlank())`
  - Label: `stringResource(stores_add_to_list_cta)` — `"Add to list"` / `"Adaugă la listă"`
  - Leading icon: `Icons.Default.Add` (20dp)
  - On tap: opens existing add-item sheet with `initialUrl = currentUrl` and `initialRegistryId = registryId` (from nav key)
  - Disabled state: button uses `MaterialTheme.colorScheme.onSurface.copy(alpha = 0.38f)` label + container via M3 default disabled styling — no override needed

**Back button behavior:**
- Device back while sheet is open: dismiss sheet first (handled by `ModalBottomSheet` default behavior)
- Device back while sheet is closed: navigate back to Store List (WebView back stack is not used for Android nav back — `webView.canGoBack()` is ignored; this keeps browsing state simple)
- Close icon in TopAppBar: always navigates back to Store List

Source: CONTEXT.md D-05 through D-16, `<specifics>` section

### 4. Add Item Screen — Registry Picker Extension

**Context:** `AddItemScreen` gains an optional `initialUrl` parameter. When entered from Store Browser, `initialUrl` is non-empty and the URL field is pre-filled; `fetchOgMetadata` triggers automatically.

**Registry picker (new UI element, shown at top of form when `registries.size >= 2`):**

Position: above the URL field, as the first element in the form `Column`.

Layout:
- Label: `Text(stringResource(stores_registry_picker_label), style = MaterialTheme.typography.labelLarge)` — `"Add to registry:"` / `"Adaugă în lista:"`
- `ExposedDropdownMenuBox` (M3 standard) with `OutlinedTextField` trigger:
  - Selected value: registry name (truncated to 1 line with `TextOverflow.Ellipsis`)
  - Dropdown items: one per registry from `RegistryPickerUiState.registries`, each a `DropdownMenuItem` with registry name
  - On selection: updates `AddItemViewModel.selectedRegistryId`
  - Default selection: `initialRegistryId` if non-null; otherwise last-used from `LastRegistryPreferencesDataStore`; otherwise first registry in list
- Hidden when `registries.size == 1` (auto-selects the single registry, no picker shown)
- `Spacer(16.dp)` between picker and URL field (md token — standard field-to-field gap)

Source: CONTEXT.md D-10, D-12

## Accessibility Contract

- All `IconButton` elements: `contentDescription` set via string resource — applies to FAB close icon, TopAppBar back/close icons
- Store card `Image` (logo): `contentDescription = "${store.name} ${stringResource(stores_logo_content_description)}"` — e.g. "eMAG Store"
- "Add to list" button: no `contentDescription` needed (label is visible text)
- Bottom bar `WindowInsets.navigationBars` padding ensures the CTA is not obscured by gesture navigation bar
- Error state "Retry" and "Back" buttons: standard `Button` / `OutlinedButton` — inherits M3 accessible touch target (48dp minimum)
- `semantics { heading() }` on the `TopAppBar` title slot (applied via `Modifier.semantics { heading() }` on the title Text) — matches Phase 2 pattern
- Registry picker `ExposedDropdownMenuBox`: M3 component handles accessibility natively (focus, announcement of options)
- FAB menu items: each `SmallFloatingActionButton` has `contentDescription` matching its label text
