Create the Android string resource directory structure and files. These establish the feature-namespaced key convention (`feature_screen_element` snake_case) that all subsequent phases must follow.

**Naming convention (per D-11 — feature-namespaced):**
- `app_` prefix: app-level strings
- `common_` prefix: shared across screens (OK, Cancel, Save, errors)
- `auth_` prefix: authentication screen strings (Phase 2 expands)
- `registry_` prefix: registry management strings (Phase 3 expands)
- `reservation_` prefix: reservation flow strings (Phase 4 expands)

**1. Create directories:**
```bash
mkdir -p app/src/main/res/values
mkdir -p app/src/main/res/values-ro
```
